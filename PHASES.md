## Assumptions / Blockers / Decisions

- **2026-06-12 ‚Äî Project location:** Built in its own repo `/Users/sahielbose/Signal-Scout` (remote `github.com/sahielbose/Signal-Scout`), fully separate from any other project per the kickoff instruction.
- **2026-06-12 ‚Äî Flat single-package, not a pnpm workspace.** BUILD_PLAN ¬ß2 says "keep it flat for a 1-day build"; one `package.json`, one `tsconfig`, shared `node_modules`. `worker/` and `mcp/` run via `tsx`. Simpler typecheck/install; revisit if it ever needs independent deploys.
- **2026-06-12 ‚Äî Local DB port 5434** (not 5432/5433) to avoid colliding with other local Postgres containers on this machine.
- **2026-06-12 ‚Äî shadcn primitives hand-rolled** (`components/ui/*`) instead of the interactive `shadcn` CLI, to keep the build non-interactive/deterministic. Same Radix + cva + Tailwind-variable approach.
- **2026-06-12 ‚Äî Embeddings:** if no remote embedder key is set, fall back to a deterministic local hash-based embedding so the prefilter still functions offline (flagged TODO to swap for a real embedder). Keeps the build runnable with zero keys.

- **2026-06-12 ‚Äî Marketing brand split.** The public marketing site (`app/(marketing)`: `/`, `/open-source`, `/privacy`, `/terms`, `/data-removal`, and the login screen) uses a warm "paper" editorial palette (cream and warm brown with a dark charcoal integrations section), scoped to a `.marketing` CSS-variable override so it does not touch the product dashboard, which keeps the dark "beacon" theme. This mirrors the information architecture of the category (typed live-feed hero, deep-research dossier panel, integrations grid) with our own brand, copy, and logo per the clean-room rule.
